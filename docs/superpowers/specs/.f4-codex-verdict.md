**3. Ratios**

**RESSALVA forte:** não chame `antecipado ÷ recebíveis` de “fração dos recebíveis antecipados” com os campos atuais. Chame de **“exposição sacada / AR aberto”**.

Fórmula honesta com schema atual:

```text
exposição_sacada = sum(saldo_devedor_informado)
AR_aberto = sum(fin_contas_receber.saldo where status aberto/parcial/vencido)
ratio = exposição_sacada / AR_aberto
```

Regras:
- Se `saldo_devedor_informado` faltar: não calcula.
- Se `AR_aberto = 0`: não calcula; mostra “base incompatível”.
- Se ratio > 100%: não clamp; isso é alerta de base misturada.
- Para ratio verdadeiro, adicione input manual `valor_bruto_recebiveis_cedidos` e flag `ar_inclui_cedidos`.

Custo:
```text
custo_run_rate_anual = saldo_devedor_informado * cet_aa
custo_pct_receita = custo_run_rate_anual / receita_liquida_TTM
```

Use **receita líquida TTM**, preferencialmente competência. Se só houver caixa, rotule como caixa. Se TTM faltar ou for zero, não degrade para chute; mostre custo absoluto e “receita insuficiente”.
